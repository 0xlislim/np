### 1.4 Peer Review Session Agenda

```
PEER REVIEW SESSION AGENDA (90 minutes)

1. Introduction (5 min)
   - Share goals for the session
   - Assign review pairs/groups

2. LinkedIn Review Round (30 min)
   - 10 min per person
   - Use LinkedIn checklist
   - Take notes on feedback

3. GitHub Review Round (30 min)
   - 10 min per person
   - Use GitHub checklist
   - Take notes on feedback

4. Group Discussion (15 min)
   - Share common issues found
   - Discuss best practices observed
   - Identify quick wins for everyone

5. Action Planning (10 min)
   - Each person lists top 3 changes to make
   - Set deadline for improvements
   - Schedule follow-up session
```

## 3. LINKEDIN RECOMMENDATIONS

### 3.1 Writing Recommendations for Peers

**Template for Zone01 Peers:**
```
I had the pleasure of working with [Name] at Zone01 Oujda, where we collaborated on several full-stack projects using Angular and Spring Boot.

[Name] consistently demonstrated [specific strength - e.g., "strong problem-solving abilities", "excellent code quality", "great communication skills"].

During our work on [specific project or situation], [he/she] [specific achievement or contribution].

I highly recommend [Name] for any [target role] position. [He/She] would be a valuable addition to any development team.
```

**Example:**
```
I had the pleasure of working with Ahmed at Zone01 Oujda, where we collaborated on several full-stack projects using Angular and Spring Boot.

Ahmed consistently demonstrated strong problem-solving abilities and a deep understanding of microservices architecture.

During our work on the 01Blog project, he implemented a robust JWT authentication system that handled 100+ concurrent users flawlessly.

I highly recommend Ahmed for any Full-Stack or DevOps position. He would be a valuable addition to any development team.
```

### 3.2 Requesting Recommendations

**Message Template:**
```
Hi [Name],

I hope you're doing well! I'm currently updating my LinkedIn profile as part of our networking practices.

Would you be willing to write a brief recommendation based on our work together at Zone01? I'd be happy to write one for you as well.

If you're open to it, you could mention:
- Our collaboration on [specific project]
- Any skills you've observed (problem-solving, code quality, teamwork, etc.)

Let me know if you'd like me to send you a recommendation first!

Thanks,
Ali
```

## 4. COLLECTIVE BRANDING

### 4.1 Zone01 Oujda Brand Elements

**Consistent Messaging:**
- Always mention "Zone01 Oujda" (not just Zone01)
- Reference "42 Network" for international recognition
- Highlight peer-learning methodology

**Hashtags to Use:**
- #Zone01Oujda
- #42Network
- #Zone01
- #PeerLearning

### 4.2 Supporting Peers' Posts

**Engagement Actions:**
1. **Like** every post from Zone01 peers
2. **Comment** thoughtfully (not just "Great post!")
3. **Share** interesting posts with your network
4. **Tag** relevant peers in related content

**Good Comment Examples:**
```
"This is a great breakdown of microservices! We used a similar approach in our 01Blog project at Zone01. The separation of concerns really helped with testing."
```

```
"Congrats on completing the project! Your implementation of WebSockets reminds me of what we learned during our real-time forum project. Keep it up!"
```

### 4.3 Cross-Promotion Template
